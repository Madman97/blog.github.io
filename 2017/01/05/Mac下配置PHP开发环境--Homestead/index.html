Mac下配置PHP开发环境--Homestead

发表于 2017-01-05 | 分类于 PHP |

一、

准备环节

—-安装Virtual Box和Vagrant

VirtualBox是Oracle的一个免费开源的虚拟机系统，和VMware比肩。由于VMware需要收费，且价格略高，所以Virtual box是大多数人的首选虚拟机。
Vagrant是配置Homestead的核心所在。Vagrant是一个基于Ruby的工具，用于创建和部署虚拟化开发环境。它 使用Oracle的开源VirtualBox虚拟化系统，使用 Chef创建自动化虚拟环境。

建议直接网上搜索下载，然后傻瓜式安装。方便快捷。

—-配置git

不出意外的话，Mac应该是自带git的(因为我的带了。。) 使用 git -v 查看是否安装了git。有了git后，输入 git config –global user.name “你的用户名” 、git config –global user.email “你的邮箱” 来全局配置git。然后，cd 到 ~/.ssh 查看是否有 id_rsa、id_rsa.pub这两个文件，如果没有的话，输入 ssh-keygen -t rsa -C “你的邮箱” 按3个回车，密码为空，来生成git密钥。

二、

下载Homestead镜像

Homestead是 laravel 封装好的一个基本完美PHP环境的镜像。由于国内网络环境原因，这里本是使用 vagrant box add laravel/homestead 安装镜像，于是乎，直接下载到本地吧。关于镜像下载地址，由于不确定网上的版本是否是最新(有时候下载版本不对也会报错~~)，所以，请先输入 vagrant box add laravel/homestead 命令来运行下载镜像，然后会弹出一个选择，选择 Virtual Box 对应的数字，然后复制下面显示的下载路径，此路径不出意外将是最新的。
下载好镜像后，使用 vagrant box add laravel/homestead “你的下载路径” 来安装镜像。不出意外的话，会出现一段successful的话，说明你安装成功了，输入 vagrant box list 可以查看你安装的虚拟机。如果安装失败查看失败原因一步步解决—-毕竟程序的世界处处是坑。

创建一个文件夹Test

在你喜欢的地方新建一个Test文件夹，比如 ~/Test 。然后在此文件夹下使用git克隆Homestead的配置文件夹，像这样：git clone https://github.com/laravel/homestead.git Homestead 。现在在Test文件夹下就存在了Homestead文件夹，cd Homestead进入，输入 bash init.sh 创建 Homestead.yaml 文件，此文件位于 ~/.homestead 目录下。输入 vim ~/.homestead/Homestead.yaml 来查看Homestead的基本配置。

创建一个文件夹firstproject

现在，在Test文件夹下与Homestead同级创建一个firstproject文件夹，此文件夹将用来存放我们的项目代码。先 cd 进入 输入 vim info.php 然后在info.php中输入<?php phpinfo(); ?>

修改配置

输入 vim ~/.homestead/Homestead.yaml 进入配置文件，找到folders项，修改：

folders:

  • map: ~/Test

to: /home/vagrant/Test
这个是配置共享文件夹，既虚拟机和本地共享的文件夹，这就实现了本地开发在虚拟机中跑项目的功能。

找到sites项，修改：

sites:

  • map: first.app

to: /home/vagrant/Test
这个是配置Nginx站点，是不是很简单，那些麻烦复杂的配置操作，Homestead都帮你做好了。如果你有多个项目的话，配置多站点，直接在sites下面仿照这两行代码一直写下去就好了。

记得修改hosts

修改好了后，输入 vim /etc/hosts 在后面加上 192.168.10.10 first.app

三、

骚年，跑起来

输入 cd ~/Test/Homestead 进入Homestead文件夹，输入 vagrant up 运行虚拟机

你可以使用 vagrant ssh 连接此虚拟机

连上虚拟机后 cd ~/vagrant 下可以查看共享文件夹，既你的项目文件夹

你可以在此虚拟机中连接数据库，用户名/密码 homestead/secret

在修改完Homestead.yaml文件后，记得在Homestead文件夹下使用 vagrant provision 使配置生效。

运行成功后，在浏览器输入 ：first.app/info.php 既能访问

如果运行失败请仔细看报错并一步步排查，遇到问题，然后一步步解决的过程才是最大的收获。
